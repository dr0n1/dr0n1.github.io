PHP无参数RCE

代码如下：

首先看这个正则[^\W]+\((?R)?\)

第一部分[^\W]：
其中[]表示匹配的开始结束，^表示取反
\W,(注意这个W是大写的)，匹配非字母、数字、下划线。等价于[^A-Za-z0-9_]
所以[^\W]是对上面的\w取反: 匹配所有字母数字下划线的字母

+是为了拼接表达式

第二部分\((?R)?\):
其中两侧的\(和\)表示匹配括号
(?R)表示递归表达式本身
(?R)?,最后的?表示匹配1个或者0个表达式本身，最后的?必不可少的

所以我们的payload必须没有参数

很容易想到通过session和hex2bin函数进行rce，但是这里session被禁用了，但是我们还可以通过传递的参数来进行RCE

在php中有个函数能返回一个包含所有已定义变量列表的多维数组
就是get_defined_vars()

利用这个函数我们就可以构造如下payload：
?code=var_dump(get_defined_vars());&a=1

会返回array(4) { ["_GET"]=> array(2) { ["code"]=> string(29) "var_dump(get_defined_vars());" ["a"]=> string(1) "1" } ["_POST"]=> array(0) { } ["_COOKIE"]=> array(0) { } ["_FILES"]=> array(0) { } }

现在我们需要将["a"]=> string(1) "1"提取出来

先使用current()函数返回数组中的当前元素的值

在使用end()函数指向最后一个元素即可

最后发现end被过滤了，问题不大，使用array_pop()函数效果是一样的

所以最后的payload为?code=eval(array_pop(current(get_defined_vars())));&a=phpinfo();
